Section 1
Why Python alongside R?

You've already learned R in Statistical Computing — so why Python? The honest answer is: the actuarial industry uses both, and increasingly leans on Python for data pipelines, machine learning, web applications, and automation.

R is a statistician's language — built for data analysis, with elegant plotting and an enormous library of statistical methods. Python is a general-purpose language that has grown very powerful statistical tools on top of a foundation that can also build web apps, connect to databases, and run on servers.

As an actuary you will move between both. This course ensures you're fluent in Python without forgetting what you know in R.

The R Bridge: Throughout this course, every concept is mapped from R to Python. You won't be starting from zero — you'll be translating. This is significantly faster than learning from scratch.

Create a new Jupyter notebook called 01_first_actuarial_notebook.ipynb and enter each cell below.

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

Line-by-line explanation — Cell 1
import numpy as np
Loads NumPy and gives it the alias np. NumPy is Python's engine for fast numerical operations — think of it as R's built-in vector arithmetic, but as a separate package. The alias means you write np.mean() instead of numpy.mean(). This alias is a universal industry convention. R: library(base) — arithmetic is built-in
import pandas as pd
Loads Pandas, your primary data manipulation tool. This is the Python equivalent of R's data.frame combined with dplyr and tidyr. The alias pd is universally used — every Pandas script in the world starts with this line. R: library(dplyr); library(tidyr)
import matplotlib.pyplot as plt
Loads the plotting module from Matplotlib. pyplot is a sub-module (hence the dot), and plt is the conventional alias. This is R's base plot() function, but much more customisable. You will see plt.show() used to display charts. R: base plot() / ggplot2
import seaborn as sns
Loads Seaborn, a statistical plotting library built on top of Matplotlib. It produces cleaner charts with less code — closer to R's ggplot2 in spirit. The alias sns is standard. R: library(ggplot2)
%matplotlib inline
A Jupyter "magic command" — the % prefix is the signal. It tells Jupyter to display all plots directly inside the notebook, beneath the cell that created them, rather than opening a separate window. This line only works inside Jupyter — not in plain Python scripts. Always include it when plotting in a notebook.
print("Libraries loaded ✓")
Confirms the imports worked. Python's print() is equivalent to R's cat() or print(). Always add a confirmation line — silent cells leave you wondering whether anything ran. R: cat("Libraries loaded\n")
print(f"Pandas version: {pd.__version__}")
An f-string (formatted string) — the f"..." syntax lets you embed variable values directly inside a string using curly braces {}. So {pd.__version__} inserts the actual installed version number, e.g. 2.1.0. The .__version__ attribute is available on most Python libraries. R: paste("Pandas version:", pd.__version__)
Cell 2Create a sample claims dataset
np.random.seed(42)
n = 200

claims = pd.DataFrame({
    'policy_id':    range(1001, 1001 + n),
    'age_band':     np.random.choice(['18-25', '26-35', '36-50', '51+'], n,
                        p=[0.15, 0.30, 0.35, 0.20]),
    'vehicle_type': np.random.choice(['Saloon', 'SUV', 'Hatchback'], n),
    'claim_amount': np.random.lognormal(mean=7.5, sigma=1.2, size=n).round(2),
    'at_fault':     np.random.choice(['Yes', 'No'], n, p=[0.4, 0.6]),
    'claim_year':   np.random.choice([2021, 2022, 2023], n)
})

missing_idx = np.random.choice(claims.index, 12, replace=False)
claims.loc[missing_idx, 'claim_amount'] = np.nan

print(f"Dataset created: {len(claims)} rows, {claims.shape[1]} columns")
Line-by-line explanation — Cell 2
np.random.seed(42)
Sets the random number seed. Any number works — 42 is just a convention. This guarantees that every time you run this cell, NumPy's random functions produce the exact same sequence of numbers. Without it, your dataset would be different every run, making analysis unreproducible. R: set.seed(42)
n = 200
Stores the dataset size in a variable. Using a variable instead of writing 200 everywhere means you can change the size in one place only — good coding practice. Avoid "magic numbers" scattered through your code.
claims = pd.DataFrame({...})
Creates a DataFrame from a Python dictionary. Each key becomes a column name, each value becomes that column's data. The entire dictionary is the argument to pd.DataFrame(). R: data.frame(col1 = ..., col2 = ...)
range(1001, 1001 + n)
Generates integers 1001, 1002, … 1200 as unique policy IDs. range(start, stop) is exclusive of the stop value — so range(1001, 1201) gives 200 values. R: seq(1001, 1200)
np.random.choice([...], n, p=[...])
Randomly picks n values from the list with weighted probabilities given by p. The probabilities must sum to 1.0. Here 15% of policyholders are aged 18–25, 30% aged 26–35, and so on — a realistic portfolio distribution. R: sample(c('18-25',...), n, replace=TRUE, prob=c(0.15,...))
np.random.choice([...], n)  # no p
Same function, no p argument — each option is equally likely. When p is omitted, NumPy assumes a uniform distribution across the choices.
np.random.lognormal(mean=7.5, sigma=1.2, size=n).round(2)
Generates n log-normally distributed values. The log-normal is fundamental in actuarial science — it models claim severities because it is always positive and right-skewed (most claims moderate, a few very large). mean and sigma are the parameters of the underlying normal distribution. The chained .round(2) rounds every value to 2 decimal places. R: rlnorm(n, meanlog=7.5, sdlog=1.2)
np.random.choice(claims.index, 12, replace=False)
Selects 12 row indices without replacement — so no row is chosen twice. claims.index is the list of row labels (0 to 199). We use these to inject missing values, simulating real-world data quality issues.
claims.loc[missing_idx, 'claim_amount'] = np.nan
.loc[rows, column] selects specific cells by label and assigns np.nan (Not a Number) — Python's missing value for numerics. np.nan is equivalent to R's NA. This is how you write to specific cells in a DataFrame. R: claims$claim_amount[missing_idx] <- NA
f"... {len(claims)} ... {claims.shape[1]} ..."
len(claims) returns the number of rows (200). claims.shape is a tuple of (rows, columns), so .shape[1] gives the column count. Printing this confirmation is a good habit — always verify your dataset was built as expected before analysis.

Line-by-line explanation — Cell 3
display(claims.head(5))
.head(5) returns the first 5 rows, just like R's head(claims, 5). We wrap it in display() rather than print() because in Jupyter, display() renders DataFrames as a formatted HTML table — much more readable than plain text. R: head(claims, 5)
claims.info()
Prints a concise diagnostic summary: column names, data types (int64, float64, object for strings), and how many non-null values each column has. This immediately reveals which columns have missing data. R: str(claims)
claims.describe().round(2)
Computes descriptive statistics for all numeric columns: count, mean, standard deviation, min, Q1 (25th percentile), median (Q2, 50th percentile), Q3 (75th percentile), and max. The chained .round(2) formats the output to 2 decimal places. R: summary(claims)
claims.isnull().sum()
A two-step chain. First, .isnull() returns a same-shaped DataFrame of True/False — True where a value is missing. Then .sum() adds up the True values per column (Python treats True as 1, False as 0). The result is the missing-value count per column — you should see 12 in claim_amount. R: colSums(is.na(claims))
"\n=== ... ==="
\n inside a string is a newline character — it adds a blank line before the section header in the output, creating visual separation between blocks. A small touch that makes notebook output much easier to read.

Line-by-line explanation — Cell 4
claims.dropna(subset=['claim_amount']).copy()
.dropna(subset=[...]) removes rows where claim_amount is missing. The subset argument restricts the check to that column only — rows with missing values in other columns are kept. The chained .copy() creates an independent copy — without it, df_clean would be a view of claims, and edits to one might affect the other. Always .copy() when creating a cleaned subset. R: df_clean <- claims[!is.na(claims$claim_amount), ]
(df_clean ... )
Outer parentheses allow the method chain to span multiple lines. Python treats everything inside matching parentheses as one expression — so you can break after each . for readability. This is the idiomatic way to write long Pandas chains. R: the pipe operator %>% serves the same purpose
.groupby('age_band')['claim_amount']
Groups the DataFrame by unique values in age_band, then selects only the claim_amount column to aggregate. This is identical to dplyr's group_by(age_band) followed by selecting a variable. R: df_clean %>% group_by(age_band)
.agg(count='count', mean_claim='mean', ...)
Applies multiple aggregations at once with custom output names. The format is new_column_name='aggregation_function'. This creates one row per age band with all five statistics as columns. Named aggregations keep output readable — not just claim_amount_mean. R: summarise(count=n(), mean_claim=mean(claim_amount), ...)
.reset_index()
Moves the group key back to a regular column. After a groupby(), the grouping column (age_band) becomes the DataFrame's index rather than a column. reset_index() converts it back so the table displays correctly. This step trips up many beginners — remember it every time you groupby.
.sort_values('mean_claim', ascending=False)
Sorts rows by mean_claim descending (highest first). ascending=False means largest values appear at the top — quickly showing which age band has the most costly claims. R: arrange(desc(mean_claim))

fig, axes = plt.subplots(1, 2, figsize=(12, 4))

Defines the correct display order for age bands. Without this, Matplotlib might sort alphabetically or arbitrarily. For labels that have a natural order (young→old, low→high), always define it explicitly — never rely on the default.
.groupby('age_band')['claim_amount'].mean().reindex(age_order)
Computes mean claim per age band, then .reindex(age_order) reorders the result to match our defined list. Without .reindex(), bars might appear in an unintended order. The result is a Pandas Series with age bands as the index and mean amounts as values.
axes[1].bar(mean_claims.index, mean_claims.values, ...)
Draws a bar chart on the right subplot. First argument is x-axis labels (mean_claims.index — age bands), second is bar heights (mean_claims.values — the numeric array). In Pandas, .index gives row labels and .values gives the raw array. R: geom_bar(stat='identity')
plt.tight_layout()
Automatically adjusts spacing between subplots so titles and labels don't overlap. Always call this before plt.show() when you have multiple subplots — without it, long axis labels frequently bleed into adjacent charts.
plt.show()

Section 1
Python's Core Data Types

Every value in Python has a type. Understanding types is essential for actuarial work — the wrong type causes silent errors (a premium stored as a string won't add correctly). Python has fewer data types than R but they behave very predictably.

Cell 1Scalar types — int, float, str, bool
# --- Integers: whole numbers ---
policy_count    = 1500
claim_year      = 2024
print(type(policy_count))          # <class 'int'>

# --- Floats: decimal numbers ---
annual_premium  = 4320.50
loss_ratio      = 0.73
interest_rate   = 0.085
print(type(annual_premium))         # <class 'float'>

# --- Strings: text ---
policy_id       = "POL-2024-00182"
insurer_name    = 'Old Mutual Life'   # single or double quotes both work
print(type(policy_id))             # <class 'str'>

# --- Booleans: True/False ---
is_at_fault     = True
has_no_claims   = False
print(type(is_at_fault))           # <class 'bool'>

# --- Check types and do arithmetic ---
print(f"Total exposure: {policy_count * annual_premium:,.2f}")
print(f"Loss ratio %:   {loss_ratio * 100:.1f}%")
Line-by-line explanation — Cell 1
policy_count = 1500
Integer assignment. Python automatically infers the type — you never declare int x = 1500 as you would in Java or C. Integers have no decimal point. Use them for counts (policies, claims, years). R: policy_count <- 1500L (or just 1500)
annual_premium = 4320.50
Float (floating-point decimal). Any number with a decimal point is a float. Premiums, rates, amounts — almost all actuarial numeric values are floats. Python floats are 64-bit doubles, same as R's default numeric type. R: annual_premium <- 4320.50
print(type(policy_count))
type() returns the data type of any variable. The output <class 'int'> confirms it's an integer. This is your debugging tool — when a calculation gives unexpected results, check the types of your variables first. R: class(policy_count)
policy_id = "POL-2024-00182"
String (text). Strings are sequences of characters enclosed in single or double quotes — both work. Use strings for IDs, names, categories. Note: "1500" is a string, not a number — you cannot do maths with it until you convert it with int() or float(). R: policy_id <- "POL-2024-00182"
is_at_fault = True
Boolean. Exactly two values: True or False — capitalised. Booleans are the result of comparisons (x > 0 returns True or False) and are used in conditionals. Note: True == 1 and False == 0 in Python arithmetic. R: TRUE / FALSE
f"Total exposure: {policy_count * annual_premium:,.2f}"
F-string with format specifier. Inside {} you can include a colon followed by a format code: ,.2f means "use comma thousands separators and 2 decimal places". So 6,480,750 would display as 6,480,750.00. :.1f means 1 decimal place. Format codes are very useful for financial outputs.

# --- List: ordered, mutable (changeable) ---
premiums        = [1200.00, 3450.00, 875.50, 2100.00]

age_bands       = ('18-25', '26-35', '36-50', '51+')
risk_levels     = ('Low', 'Medium', 'High')

policy = {

Line-by-line explanation — Cell 2
premiums = [1200.00, 3450.00, ...]
List — the most used collection. Square brackets, comma-separated values. A list is ordered (items stay in the order you put them), mutable (you can add, remove, change items), and can hold mixed types (though mixing is generally bad practice). R: premiums <- c(1200.00, 3450.00, ...)
premiums.append(1550.00)
Adds one item to the end of the list. This modifies the list in-place — no need to reassign. .append() is the standard way to grow a list one item at a time. R: premiums <- c(premiums, 1550.00)
premiums[0]   premiums[-1]
Indexing is zero-based in Python — the first item is at index 0, not 1. This is the most common adjustment from R (where indexing starts at 1). premiums[-1] is a shortcut for the last element — negative indices count from the end. R: premiums[1] for first, premiums[length(premiums)] for last
age_bands = ('18-25', '26-35', ...)
Tuple — immutable list. Round brackets instead of square. Once created, a tuple cannot be changed — no appending, removing, or replacing items. Use tuples for fixed reference data that should never change: age band labels, risk category names, regulatory thresholds. Immutability is a safety feature. R: no direct equivalent — use a locked vector
policy = {'id': 'POL-001', 'premium': 2400.00, ...}
Dictionary — key-value store. Curly braces with key: value pairs. Keys must be unique strings (or numbers). This is R's named list. Dictionaries are ideal for representing one record (one policy, one claim) with named attributes. R: policy <- list(id='POL-001', premium=2400.00, ...)
policy['id']   policy['premium']
Access a dictionary value by key using square brackets with the key name as a string. This is the standard access pattern — policy.id does not work for plain dictionaries (it works for objects/classes). R: policy$id or policy[['id']]
policy['no_claims_discount'] = 0.30

Control flow determines what your code does based on data values. In actuarial work this means: rating factors, benefit triggers, iteration over policies. Python's syntax is minimal — indentation (4 spaces) replaces the curly braces you're used to in R.

Writing Actuarial Functions

Functions are the building blocks of reusable actuarial code. Every formula you use repeatedly — net premium, present value, loss ratio — should become a function. Python functions are defined with def, and well-written functions have docstrings explaining what they do.

Cell 5Defining and using actuarial functions
# --- Basic function ---
def loss_ratio(claims_paid, premiums_earned):
    """
    Calculate the loss ratio for an insurance portfolio.

    Parameters:
        claims_paid (float):     Total claims paid in the period.
        premiums_earned (float): Total premiums earned in the period.

    Returns:
        float: Loss ratio as a decimal (e.g. 0.73 = 73%).
    """
    return claims_paid / premiums_earned

# --- Function with a default argument ---
def present_value(future_value, rate, years, frequency=1):
    """
    Compute the present value of a future cash flow.

    Parameters:
        future_value (float): The amount to be received in the future.
        rate (float):         Annual interest rate as a decimal.
        years (int):          Number of years until the payment.
        frequency (int):      Compounding frequency per year (default: 1 = annual).

    Returns:
        float: Present value of the cash flow.
    """
    return future_value / (1 + rate / frequency) ** (years * frequency)

# --- Function returning multiple values ---
def combined_ratio(claims, expenses, premiums):
    """Return loss ratio, expense ratio, and combined ratio."""
    lr = claims  / premiums
    er = expenses / premiums
    cr = lr + er
    return lr, er, cr

# --- Calling the functions ---
lr = loss_ratio(7_300_000, 10_000_000)
print(f"Loss ratio:    {lr:.1%}")

pv = present_value(500_000, rate=0.085, years=10)
print(f"Present value: ${pv:,.2f}")

pv_monthly = present_value(500_000, rate=0.085, years=10, frequency=12)
print(f"PV (monthly):  ${pv_monthly:,.2f}")

loss_r, exp_r, comb_r = combined_ratio(7_300_000, 1_800_000, 10_000_000)
print(f"Loss: {loss_r:.1%} | Expense: {exp_r:.1%} | Combined: {comb_r:.1%}")

A docstring — a multi-line string placed immediately after the def line. It documents what the function does, its parameters, and return value. This is the standard way to document Python functions. Error Handling with try/except

Real actuarial data is messy. Division by zero, missing fields, wrong types — your code will encounter these. Instead of crashing, use try/except to handle errors gracefully and log what went wrong.

Cell 6try / except — robust actuarial functions
def safe_loss_ratio(claims_paid, premiums_earned):
    """Calculate loss ratio with error handling for zero/invalid premiums."""
    try:
        if premiums_earned <= 0:
            raise ValueError(f"Premiums must be positive, got {premiums_earned}")
        result = claims_paid / premiums_earned
        return result
    except ZeroDivisionError:
        print("Error: Cannot divide by zero — premiums_earned is 0.")
        return None
    except ValueError as e:
        print(f"Error: {e}")
        return None
    except TypeError:
        print("Error: Both arguments must be numbers.")
        return None

# --- Test all error cases ---
print(safe_loss_ratio(7_300_000, 10_000_000))  # Normal: 0.73
print(safe_loss_ratio(7_300_000, 0))            # ZeroDivision
print(safe_loss_ratio(7_300_000, -5000))        # ValueError
print(safe_loss_ratio("lots", 10_000))          # TypeError
Line-by-line explanation — Cell 6
try:
Opens the "attempt" block. Python runs the indented code inside try. If any line raises an error (exception), Python immediately jumps to the matching except block instead of crashing the entire program. Code after the error line inside try is skipped. R: tryCatch({ ... }, error = function(e) { ... })
raise ValueError(f"Premiums must be positive...")
raise deliberately triggers an error. You use this to enforce business rules — premiums cannot be zero or negative. ValueError is the appropriate error type for invalid input values. The string message describes exactly what went wrong, which helps whoever is debugging.
except ZeroDivisionError:
Catches a specific error type. ZeroDivisionError fires when Python tries to divide by zero. By catching specific errors rather than a generic except:, you handle different failure modes differently — division by zero is a different problem from a wrong data type. R: tryCatch(..., error = function(e) { ... })
except ValueError as e:
as e captures the error object — the actual exception — into the variable e. You can then print e to see the error message. This is how you log what went wrong while still recovering gracefully. The message we passed to raise ValueError(...) appears here.
except TypeError:
Catches a TypeError — which fires when an operation is applied to the wrong type (like dividing a string by a number). In actuarial pipelines, wrong types often come from reading CSVs where numeric columns were imported as strings. Catching this lets you log the problem rather than crash.
return None
Returns None on failure. This signals "no valid result" to the caller. The caller can check if result is None: to handle failure cases. Returning None is preferable to letting the function crash, especially when processing many policies in a loop — one bad record shouldn't stop the entire run.

Section 1
NumPy — Fast Vectorised Computation

NumPy's key idea is the array — a block of numbers stored contiguously in memory. Operations on arrays apply to all elements at once without writing a loop. In R, all arithmetic is already vectorised by default — NumPy brings the same behaviour to Python.

For actuarial work this matters enormously: computing premiums for 100,000 policies, discounting cash flows for 40 years, or simulating 1,000,000 claim scenarios — all need fast vectorised computation.

Cell 1NumPy arrays and vectorised arithmetic
import numpy as np

sum_insured  = np.array([50000, 120000, 75000, 200000, 95000])
rate_per_mil = np.array([4.20,   3.85,   4.50,   3.60,   4.10])

premiums = (sum_insured / 1000) * rate_per_mil
print("Premiums:    ", premiums.round(2))

discount_factors = 1 / (1.075 ** np.arange(1, 11))
print("Disc. factors:", discount_factors.round(4))

cash_flows = np.full(10, 50000.0)
pv_annuity = np.sum(cash_flows * discount_factors)
print(f"PV of annuity: ${pv_annuity:,.2f}")

print("\n--- Array statistics ---")
print(f"Mean premium:   ${np.mean(premiums):.2f}")
print(f"Max premium:    ${np.max(premiums):.2f}")
print(f"Total exposure: ${np.sum(sum_insured):,}")
print(f"Std deviation:  ${np.std(premiums, ddof=1):.2f}")

high_value = sum_insured[sum_insured > 90000]
print(f"\nHigh-value policies (>90k): {high_value}")
Line-by-line explanation — Cell 1
np.array([50000, 120000, ...])
Creates a NumPy array from a Python list. Unlike a Python list, a NumPy array holds elements of a single type (all floats or all ints), which allows extremely fast bulk operations. Think of it as R's atomic vector. R: sum_insured <- c(50000, 120000, ...)
premiums = (sum_insured / 1000) * rate_per_mil
Vectorised arithmetic — this single line divides every element of sum_insured by 1000, then multiplies each result by the corresponding element of rate_per_mil, producing a new array of 5 premiums. No loop needed. This is identical to R's default behaviour. Python lists do not support this — only NumPy arrays do. R: premiums <- (sum_insured / 1000) * rate_per_mil
np.arange(1, 11)
Generates an integer array [1, 2, 3, …, 10]. arange(start, stop) is like Python's range() but returns a NumPy array instead of a range object — so you can do maths on it directly. Here it creates the year indices for discounting. R: 1:10
1 / (1.075 ** np.arange(1, 11))
Computes 10 discount factors at once — v¹, v², …, v¹⁰ where v = 1/1.075. The exponentiation ** applies element-wise across the array. This is the actuarial discounting pattern: discount_factor[t] = (1+i)^-t. Doing this for 40 years would just change 11 to 41 — the code doesn't change. R: 1 / (1.075 ^ (1:10))
np.full(10, 50000.0)
Creates an array of 10 identical values (50,000 each) — representing a level annuity paying $50,000 per year. np.full(size, fill_value) is the efficient way to initialise a constant array. For non-level cash flows you'd use np.array([...]) directly. R: rep(50000, 10)
np.sum(cash_flows * discount_factors)
The present value formula: multiply each cash flow by its discount factor, then sum all products. This is Σ(CF_t × v^t) — the fundamental actuarial present value calculation — done in one line on arrays of any length. R: sum(cash_flows * discount_factors)
np.std(premiums, ddof=1)
ddof=1 is critical — it applies the Bessel correction (divides by n−1 instead of n), giving the sample standard deviation rather than the population standard deviation. Always use ddof=1 for sample data. Without it, NumPy uses population std (divides by n), which is different from R's sd(). R: sd(premiums) — R uses ddof=1 by default
sum_insured[sum_insured > 90000]
Boolean indexing — sum_insured > 90000 creates a boolean array [False, True, False, True, True]. Using it as an index inside [] returns only the elements where the condition is True. This is one of the most-used patterns in NumPy and Pandas. R: sum_insured[sum_insured > 90000]
Cell 2Random simulation — Monte Carlo claims
np.random.seed(42)

n_simulations = 100_000
claim_freq    = 0.08
mean_severity = 5_000
sigma_severity= 1.2

has_claim   = np.random.binomial(1, claim_freq, n_simulations).astype(bool)
n_claims    = has_claim.sum()

severities  = np.random.lognormal(
    mean=np.log(mean_severity) - 0.5 * sigma_severity**2,
    sigma=sigma_severity,
    size=n_claims
)

total_losses = np.zeros(n_simulations)
total_losses[has_claim] = severities

print(f"Simulations:       {n_simulations:,}")
print(f"Claims occurred:   {n_claims:,} ({n_claims/n_simulations:.1%})")
print(f"Mean total loss:   ${np.mean(total_losses):,.2f}")
print(f"95th percentile:   ${np.percentile(total_losses, 95):,.2f}")
print(f"99th percentile:   ${np.percentile(total_losses, 99):,.2f}")
print(f"Prob(loss > 20k):  {np.mean(total_losses > 20_000):.2%}")
Line-by-line explanation — Cell 2
np.random.binomial(1, claim_freq, n_simulations)
Simulates claim occurrence for each of the 100,000 policies. Each policy independently has an 8% chance of a claim (p=0.08). binomial(1, p, n) is equivalent to n independent Bernoulli trials — each returns 0 (no claim) or 1 (claim). This is your first actuarial Monte Carlo. R: rbinom(n_simulations, 1, claim_freq)
.astype(bool)
Converts integers (0/1) to booleans (False/True). This lets us use the array as a boolean mask in later indexing. .astype() converts an array from one type to another without changing values. R: as.logical(...)
np.log(mean_severity) - 0.5 * sigma_severity**2
Converts mean and sigma to log-normal parameters. If X ~ LogNormal(μ, σ²) then E[X] = exp(μ + σ²/2). Rearranging: μ = log(E[X]) − σ²/2. This ensures the simulated severities have the intended mean of $5,000, not the raw log-scale mean. This is an important actuarial detail — always parameterise correctly.
size=n_claims
Only generates severities for policies that had a claim. We simulate the exact number of claims (n_claims), not one per policy. This is more efficient and accurate — policies with no claim have zero loss, handled separately below.
total_losses = np.zeros(n_simulations)
Initialises an array of zeros — all 100,000 policies start with zero loss. np.zeros(n) is the standard way to create an empty result array before populating it. R: total_losses <- rep(0, n_simulations)
total_losses[has_claim] = severities
Inserts claim severities into the positions where claims occurred. has_claim is a boolean array — it selects exactly the positions corresponding to policies that had a claim, and assigns the corresponding severity to each. Policies with no claim remain at zero.
np.percentile(total_losses, 95)
Computes the 95th percentile — the value below which 95% of simulated losses fall. In actuarial science this is a Value at Risk (VaR) estimate. np.percentile(arr, q) takes the array and the percentile level (0–100). R: quantile(total_losses, 0.95)
np.mean(total_losses > 20_000)
Estimates the probability that total loss exceeds $20,000. total_losses > 20_000 creates a boolean array; np.mean() of a boolean array gives the proportion of True values — i.e. the empirical probability. This pattern (mean of boolean) is one of the most useful tricks in actuarial simulation. R: mean(total_losses > 20000)

Line-by-line explanation — Cell 4
df_clean = df.copy()
Always copy before cleaning. Working on a copy preserves the original raw data. If you make a mistake during cleaning, you can restart from df without re-reading the file. This is a professional hygiene rule — never mutate your raw import.
df_clean['age'].fillna(df_clean['age'].median())
Imputes missing ages with the median. .fillna(value) replaces all NaNs in a Series with the given value. The median is preferred over the mean for skewed variables because it is not distorted by outliers. Other strategies: fill with mean, fill with mode (most common), or fill using a group-level median. R: df$age[is.na(df$age)] <- median(df$age, na.rm=TRUE)
.astype(int)
Converts the column to integer type. After imputing the median (which is a float), the age column becomes float64. Since ages are whole numbers, we convert back to int. Note: this only works after all NaN values are filled — you cannot store NaN in an integer column in older Pandas versions. R: as.integer(df$age)

Replaces invalid sentinel values (−99) with NaN. Negative premiums are impossible — they indicate data entry errors. We convert them to NaN first so fillna() can handle them in the next line. The pattern .loc[condition, column] = value is the standard way to conditionally update a column. R: df$premium[df$premium < 0] <- NA

Removes duplicate rows based on policy_id. Each policy should appear once — duplicates indicate data pipeline errors (e.g. a file appended twice). subset specifies which columns define a "duplicate" — here, same policy_id means same policy regardless of other column values. R: distinct(df, policy_id, .keep_all=TRUE)
df_clean['premium'].quantile(0.01) / .quantile(0.99)
Computes the 1st and 99th percentile of the premium distribution. These become the bounds for capping outliers. The .quantile(q) method takes a value between 0 and 1. R: quantile(df$premium, 0.01)

Caps (Winsorises) extreme values — values below q_low are set to q_low, values above q_high are set to q_high. Winsorisation is a standard actuarial technique for handling outliers in premium and claims data without discarding them entirely. R: pmin(pmax(df$premium, q_low), q_high)

Each bin includes the right endpoint: (0,25] means 0 < age ≤ 25. R: cut(df$age, breaks=c(0,25,35,50,65,100), labels=c(...))

Creates a boolean column — True for high-value policies, False otherwise. Boolean columns are useful as binary flags (indicators) in regression models and for quick subset filtering. The comparison operator >= applied to a column returns a boolean Series. R: df$high_value <- df$sum_insured >= 150000

np.random.choice(df_clean['policy_id'], n_claims, replace=True)
Randomly samples policy IDs from the clean dataset with replacement, assigning each claim to an existing policy. replace=True means the same policy can appear multiple times (realistic — one policy can have multiple claims). R: sample(df_clean$policy_id, n_claims, replace=TRUE)
pd.merge(claims, df_clean[['policy_id','region',...]], on='policy_id', how='left')
Joins the claims table to the policies table on the shared key policy_id. Builds a loss development triangle. Line-by-line explanation — Cell 5
import plotly.graph_objects as go
Loads Plotly's low-level graph objects module. This gives you fine-grained control over every chart element — equivalent to using raw Matplotlib rather than Seaborn. The alias go is standard. Plotly also has a high-level plotly.express module for quick charts — we use graph_objects for full control.
make_subplots(rows=1, cols=2, subplot_titles=[...])
Creates a Plotly figure with multiple subplots. Returns a figure object that traces are added to. subplot_titles sets the title for each panel. Plotly's subplot system works similarly to Matplotlib's but produces interactive output.
for product, color in colors.items():
Iterates over a dictionary's key-value pairs. .items() returns each (key, value) tuple — here (product_name, hex_color). This pattern is the standard way to loop over a dictionary when you need both the key and value simultaneously. R: for(nm in names(colors)) { ... }
fig.add_trace(go.Scatter(...), row=1, col=1)
Adds a trace (data series) to the figure. go.Scatter draws a line/point chart. row and col specify which subplot to place it in. Each call to add_trace() adds one series — we loop over products to add three lines to the left panel and three bars to the right.
hovertemplate=f"<b>{product}</b><br>Year: %{{x}}..."
Customises the hover tooltip. %{x} and %{y:.2f} are Plotly template variables — they insert the actual x and y values at the cursor position. The f-string interpolates {product} at plot-creation time. <extra></extra> removes the default trace name box. Good hover templates make interactive charts far more useful.
barmode='stack'
Stacks bar chart segments for the right panel, showing total premium with product breakdown visible within each bar. Options are 'group' (side-by-side) or 'stack'. Stacked bars are excellent for showing composition over time — a core actuarial visualisation.
hovermode='x unified'
Shows all series values in one tooltip when hovering over any x position. Instead of separate tooltips per line, a single panel appears listing all products' values for that year. Very useful for comparing multiple series — standard for financial trend charts.
fig.write_html('portfolio_interactive.html')
Saves the chart as a standalone HTML file. This file can be opened in any browser, shared by email, or embedded in a web page — no Python needed to view it. This is how you deliver interactive actuarial charts to clients or management. R: htmlwidgets::saveWidget()
